Why This Agent Exists — The Core Problem

The biggest reconciliation challenge in bank capital reporting is the gap between IFRS financial statements (what the bank reports to investors) and regulatory capital (what OSFI cares about). These two views of equity can differ by $1–3 billion at a major Canadian bank — and every dollar of difference must be explained, line by line, to OSFI in the BCAR return.

🏛 What OSFI Requires — BCAR Ch. 2
OSFI mandates a full reconciliation between the bank's published IFRS balance sheet and the BCAR regulatory capital return (Schedules 1 and 3). Every line of the BCAR must tie back to:

① A specific GL account or set of accounts
② A documented regulatory adjustment (with basis in OSFI CAR)
③ A CFO-attested certification of accuracy
④ An audit trail satisfying OSFI's Model Risk (E-23) and internal controls expectations

Failure to reconcile = OSFI supervisory letter + potential D-SIB buffer add-on.
⚙ What the Agent Does — In Plain English
The GL-to-Capital Reconciliation Agent automatically bridges three systems that historically never spoke to each other:

📒 GL (Oracle / SAP) — Book equity, goodwill, DTA balances
🏦 Capital Engine — IRB EL shortfall, CVA, SFT exposures
📊 BCAR Portal — OSFI-required return format

It applies all 15 OSFI regulatory deductions in sequence, maintains data lineage for each adjustment, flags discrepancies to the Capital team, and produces the OSFI-ready workpaper with CFO sign-off workflow — in T-5 calendar days, not T-15.

The 15 OSFI Regulatory Deductions — Agent Calculation Logic

Each deduction is a distinct calculation challenge requiring GL account mapping, regulatory interpretation, and in several cases, inputs from the Capital Engine (IRB models). The agent automates all 15, with thresholds, phase-in schedules, and cross-checks hard-coded to OSFI CAR 2024.

Break Analysis — What Happens When the Reconciliation Doesn't Balance

The agent classifies every break by severity, type, and action path — from automatic self-correction to immediate CRO escalation. Based on the reconciliation skill's break categorization methodology.

🔴 P1 — CRITICAL BREAK
Total Capital Ratio below OSFI floor
CET1 falls below 11.0% (OSFI combined minimum). Could trigger OSFI supervisory intervention or public disclosure requirement under OSFI Pillar 2 guidance.
Agent Action: Immediate alert to CRO, CFO, CEO. Suspend BCAR submission. Escalate to BCAR Orchestrator with 2-hour response SLA. Activate break-glass capital action playbook.
🔴 P1 — CRITICAL BREAK
GL to BCAR Unexplained Difference >$100M
Automated check: sum of all 23 waterfall items ≠ BCAR Schedule 3 Line 14. Gap of $125M with no reconciling item ID.
Agent Action: Lock submission. Re-pull all GL data. Compare hash of source data to prior run. Assign to Capital team lead with 4-hour SLA. Log in RAID register.
🟡 P2 — HIGH BREAK
Goodwill DTL Offset Change >$50M QoQ
DTL associated with goodwill dropped from $420M to $370M — unexpected given no impairment or disposal. Agent flags as unexplained variance requiring Accounting commentary.
Agent Action: Auto-draft commentary based on GL movement analysis. Assign to Accounting team with 48-hr SLA. Require sign-off from Chief Accountant before CRO review proceeds.
🟡 P2 — HIGH BREAK
IRB EL Shortfall Sign Change
Prior quarter: EL < provision (surplus). This quarter: EL > provision (shortfall). Switches from T2 credit to CET1 deduction — material capital impact.
Agent Action: Alert Capital Calculation Agent and Model Risk. Request IRB model run explanation. Draft board note on provision adequacy. Escalate to CRO for sign-off before closing.
🟢 P3 — ROUTINE BREAK
Timing Difference — GL Period vs. Capital Engine
IRB model runs on T-9; GL closes on T-15. $12M difference in EAD for revolving credit facilities due to daily draw activity between dates.
Agent Action: Auto-document as timing difference. Cross-reference to Capital Engine run date. Apply standard reconciling item treatment. Auto-resolve if within ±$25M tolerance.
🟢 P3 — ROUTINE BREAK
Rounding / FX Translation Difference
$4.2M difference from USD/CAD translation of US subsidiary goodwill — agent uses quarter-end rate; BCAR system uses monthly average.
Agent Action: Auto-post standard FX reconciling item. Document exchange rate used. Confirm with Treasury for rate consistency. No escalation required if <$10M.

Controls Framework — SOD, Audit Trail & OSFI E-23 Compliance

Grounded in the si-security-controls skill. Every capital figure produced by this agent carries an immutable provenance chain: calculated-by, reviewed-by, approved-by — satisfying both OSFI's supervisory expectations and Internal Audit requirements.

🔐 Segregation of Duties (SOD) Matrix
  • 🟦
    Role: GL Data Extractor — pulls trial balance from Oracle/SAP. Cannot modify GL accounts or approve capital figures.
  • 🟩
    Role: Deduction Calculator (Agent) — applies all 15 deductions autonomously. Cannot modify source GL data or approve results.
  • 🟨
    Role: Capital Controller — reviews and challenges calculations. Cannot modify GL or recalculate deductions directly.
  • 🟥
    Role: CRO — approves reconciliation. No system access to GL or capital engine. Pure governance/oversight.
  • ⬛
    Role: CFO — BCAR attestation only. Receives final ratios + workpaper. Zero access to calculation inputs.
  • 🔵
    SOD Conflict Rule: No individual can both (a) post a GL journal in an equity/capital account AND (b) approve the capital ratio derived from that account.
📋 Audit Trail — Every Calculation Logged
  • →
    Immutable Log: Every agent calculation is written to an append-only audit log (Splunk / Azure Sentinel) with: timestamp, input data hash, calculation version, result, and agent run ID.
  • →
    Data Lineage: Each BCAR line item traces back to: source GL account → extract timestamp → deduction formula version → OSFI CAR section reference.
  • →
    Version Control: All calculation rules (OSFI CAR parameters) are versioned in Git. Quarterly updates to risk weights, thresholds, and phase-in schedules are reviewed and approved before deployment.
  • →
    Retention: All workpapers retained 7 years per OSFI examination requirements. Available for on-demand OSFI review.
  • →
    Non-repudiation: Digital signatures on all approvals (CRO, CFO). Timestamps from certified time authority.
🔬 OSFI E-23 Model Risk Compliance
  • →
    Model Inventory: Each deduction calculation is catalogued in the model risk inventory with: purpose, owner, validation date, limitations, compensating controls.
  • →
    Threshold Calculations: The 10% and 17.65% threshold tests (D7, D11, D12) are subject to independent model validation annually per OSFI E-23 §3.2.
  • →
    IRB EL Model: The EL shortfall calculation (D10) uses OSFI-approved IRB models. Model change log maintained. Material model updates require OSFI notification under CAR §A.3.
  • →
    Override Controls: Any manual override to an agent-calculated deduction requires dual sign-off (CRO + Chief Accounting Officer) and is flagged in the BCAR submission cover note.

  • ✓
